# Build the Next.js Frontend

The backend architecture has been completed.

I now want to build the frontend described in Assignment 3.

The frontend should consume the existing Rust backend and demonstrate secure API integration.

Do not redesign the backend.

Build the frontend around the existing APIs.

---

# Technology Stack

Use:

* Next.js (App Router)
* TypeScript
* React
* Tailwind CSS
* React Query (TanStack Query)
* Axios
* Zod
* React Hook Form

Enable strict TypeScript mode.

Follow modern Next.js best practices.

---

# Goal

This frontend is intended to demonstrate:

* Server Side Rendering (SSR)
* Secure API communication
* Clean component architecture
* Type safety
* Reusable UI
* Production-ready project organization

The UI should be simple and professional.

No unnecessary animations.

---

# Project Structure

Organize the application using feature-based modules.

Example:

```text
app/

    inventory/

    certificates/

components/

    dashboard/

    certificates/

    ui/

lib/

    api/

    hooks/

    utils/

types/

services/
```

Avoid placing everything inside one folder.

---

# Environment Variables

Use environment variables for API URLs.

Example:

```
NEXT_PUBLIC_CERTIFICATE_API=

NEXT_PUBLIC_INVENTORY_API=
```

Never hardcode URLs.

---

# Inventory Page

Implement:

```
/inventory
```

This page should be rendered using **Server Side Rendering (SSR)**.

Fetch certificate data from the Inventory Service.

Display:

* Certificate Subject
* Issuer
* Status
* Expiration Date
