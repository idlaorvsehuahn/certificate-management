# Arkion Certificate Management Platform — Web Client

This is a Next.js (App Router) web interface styled with Arkion's premium dark identity governance theme. It integrates with the Certificate Service (write-model) and Inventory Service (read-model) to manage machine, AI agent, and service account identities.

---

## Features

1. **Non-Human Identity (NHI) Ledger**: Live queryable inventory table supporting search by Common Name (Subject) or Issuer Authority, status filtering (`Active`, `Revoked`, `Expired`), and pagination.
2. **Dashboard Overview**: Microservice metrics dashboard showing Total registered identities, Active trust state count, Revoked certificates, and credentials expiring within 30 days.
3. **Identity Detail View**: Fully detailed view of certificate details including trust timeline, subject alternative names (SANs), and revocation reason warning panels for revoked certs.
4. **On-Demand Provisioning**: React Hook Form + Zod validated form interface to issue new X.509 certificates to the local PKI, rendering the private credentials for download/copy on success.
5. **Modern Architecture**: Leverages Server-Side Rendering (SSR) for initial load speeds combined with TanStack Query caching for fluid client-side searching and filters.

---

## Tech Stack

- **Framework**: Next.js 15 (App Router)
- **Styling**: Tailwind CSS v4 (using `@theme` custom design properties)
- **State/Caching**: `@tanstack/react-query` (TanStack Query)
- **Forms & Validation**: `react-hook-form` + `zod` schema resolvers
- **Icons**: `lucide-react`
- **Network**: `axios`

## Getting Started

### 1. Configure Environment Variables
The application uses `.env.local` to resolve the APIs for both microservices. Ensure `frontend/.env.local` contains:
```env
NEXT_PUBLIC_CERTIFICATE_API=https://localhost:8080
NEXT_PUBLIC_INVENTORY_API=https://localhost:8081
```

### 2. Install Dependencies
Run from the `frontend` directory:
```bash
npm install
```

### 3. Run the Development Server
```bash
npm run dev
```
Open [http://localhost:3000](http://localhost:3000) with your browser to access the platform.

### 4. Build for Production
Verify that typescript types and compiler packages build cleanly:
```bash
npm run build
```

## TLS & Self-Signed Certificates in Dev

During local development, both Rust microservices listen on HTTPS using self-signed development certificates generated by the PKI tool (`tools/certgen`).

To prevent Node.js from throwing security rejection errors during Server-Side Rendering (SSR) fetches, the backend Axios client utilizes a custom HTTPS agent in server contexts:
```typescript
const httpsAgent = typeof window === 'undefined'
  ? new https.Agent({ rejectUnauthorized: false })
  : undefined;
```
This allows Server Components to safely execute pre-fetches of credentials. For browser context requests, trust is managed by accepting the self-signed Root CA (`ca.crt`) in your local system keychain.
